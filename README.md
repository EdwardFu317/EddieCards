# EddieCards

EddieCards 是一个面向 Obsidian 的闪卡与复习插件。它把 Markdown 笔记、卡组、微卡组、有序学习队列、AI 制卡和 Anki 风格导出整合在一起，目标是在 Obsidian 内完成从笔记到复习的闭环。

插件名：`EddieCards`  
插件 id：`ob-ki`  
最低 Obsidian 版本：`1.5.0`

## 核心功能

- 从选中的 Markdown 文本快速制卡。
- 支持基础卡和习题卡两种卡片类型。
- 使用 FSRS 进行普通复习调度。
- 支持提前复习、全量复习、挂起全量复习，且这些模式不计入 FSRS。
- 支持卡组、卡组文件夹、微卡组、嵌套微卡组。
- 支持自定义学习顺序：卡片和微卡组可以拖拽排序。
- 支持把卡片拖进微卡组。
- 支持卡片挂起，也支持从微卡组右键批量挂起其中卡片。
- 支持 AI 从当前笔记生成闪卡，并在加入前预览、勾选。
- 支持导出为 Anki `.apkg`。
- 支持完整数据备份和恢复。
- 支持来源笔记跳转、双链跳转、健康检查。

## 打开 EddieCards

可以通过以下方式打开插件界面：

- 点击左侧 Ribbon 的 EddieCards 图标。
- 使用命令面板：`打开 EddieCards 卡组界面`。
- 快捷制卡后，如果设置中开启“添加卡片后打开卡组界面”，会自动打开 EddieCards。

卡组右键菜单包含：

## 制卡

### 基础卡

在 Markdown 编辑器中选中一段内容，按基础卡制卡快捷键，选中的 Markdown 会作为卡片背面。

创建窗口中需要填写：

- 卡组。
- 微卡组，可选择卡组顶层、已有微卡组，或新建微卡组。
- 卡片类型：基础。
- 正面问题。

默认基础卡快捷键：

```text
Mod+Shift+A
```

`Mod` 在 Windows/Linux 中是 `Ctrl`，在 macOS 中是 `Command`。

### 习题卡

习题卡用于把同一段选中文本切成“题目”和“解答”。

操作方式：

- 选中一段 Markdown。
- 按习题卡制卡快捷键。
- 在弹窗里拖动竖向分割线。
- 分割线前面的内容作为正面题目。
- 分割线后面的内容作为背面解答。

默认习题卡快捷键：

```text
Mod+Shift+E
```

### 制卡时新建微卡组

在制卡弹窗的“微卡组”下拉菜单中可以选择：

```text
+ 新建微卡组
```

选择后可以同时填写：

- 新微卡组名称。
- 新微卡组位置。

位置可以是：

- 卡组顶层。
- 某个已有微卡组下面。

保存时插件会先创建微卡组，再把新卡片加入这个微卡组。

## 微卡组

微卡组是卡组内部的有序文件夹。它可以用于表达学习顺序，例如先概念、后定理、再习题。

支持：

- 选择多张卡组成微卡组。
- 在微卡组内继续组成子微卡组。
- 无限嵌套。
- 微卡组重命名。
- 添加子微卡组，并自动进入命名状态。
- 微卡组折叠和展开。
- 微卡组移动到其他卡组或其他微卡组。
- 解散微卡组。
- 删除微卡组。
- 从上级微卡组移出。

默认组成微卡组快捷键：

```text
Mod+Shift+M
```

## 学习顺序

每个卡组可以选择是否启用自定义排序。

启用后：

- 初始复习队列会优先按你排好的卡片和微卡组顺序生成。
- 同级卡片和微卡组可以拖拽调整顺序。
- 卡片可以拖到微卡组行上，直接移入该微卡组。
- 微卡组可以嵌套进其他微卡组。

关闭后：

- 普通复习主要按 FSRS 到期时间排序。

注意：进入复习后，具体队列推进仍按当前复习队列处理，不会因为某张卡答错就强行回到全局自定义排序顶部。

## 自定义顺序调度

EddieCards 的自定义顺序调度用于解决“知识前置关系”问题。例如还没有复习概念时，不希望系统先推送依赖它的定理或习题。

每个卡组都可以单独设置：

- 启用自定义排序。
- 关闭自定义排序。

开启后，复习队列的初始生成会优先参考卡片与微卡组树中的顺序：

```text
微卡组 1
  卡片 A
  卡片 B
  子微卡组
    卡片 C
微卡组 2
  卡片 D
未分类卡片 E
```

插件会按这个树形顺序展开卡片，先得到一个“学习顺序”，再结合复习模式筛选实际要复习的卡片。

不同模式下的行为：

- 普通复习：先筛出已到期且未挂起的卡，再按自定义顺序排列；同等情况下仍参考 FSRS 到期信息。
- 提前复习：复习未挂起卡，按自定义顺序排列，不计入 FSRS。
- 全量复习：复习所有未挂起卡，按自定义顺序排列，不计入 FSRS。
- 挂起全量复习：复习全部卡片，包括挂起卡，按自定义顺序排列，不计入 FSRS。
- 微卡组复习：只在该微卡组范围内展开顺序。

关闭后，普通复习会更接近传统 Anki 风格，主要按 FSRS 到期时间排序。

自定义顺序调度不会替代 FSRS。它更像是在 FSRS 选出候选卡之后，给这些卡增加一个“知识结构优先级”。

## 选择与批量操作

卡片列表支持：

- 勾选单张卡。
- 批量选择。
- Shift 连选。
- 全选当前筛选结果。

批量操作包含：

- 组成微卡组。
- 加入指定微卡组。
- 移动到卡组或微卡组。
- 挂起选中卡片。
- 取消挂起选中卡片。
- 重置学习进度。
- 删除选中卡片。

## 卡片右键菜单

单张卡片右键支持：

- 查看详情。
- 编辑卡片。
- 打开来源。
- 加入选择或取消选择。
- 用选中卡片组成微卡组。
- 加入指定微卡组。
- 移动到卡组或微卡组。
- 挂起卡片或取消挂起卡片。
- 重置学习进度。
- 删除卡片。

如果右键的是已选中的卡片，部分操作会自动作用于当前选中的多张卡。

## 微卡组右键菜单

微卡组右键支持：

- 重命名微卡组。
- 添加子微卡组。
- 移动到卡组或微卡组。
- 挂起此微卡组。
- 取消挂起此微卡组。
- 复习此微卡组。
- 全量复习此微卡组。
- 挂起全量复习此微卡组。
- 移出上级微卡组。
- 解散微卡组。
- 删除微卡组。

“挂起此微卡组”本质上是递归挂起其中的所有卡片，不会给微卡组本身单独增加一种复杂状态。

## 复习模式

### 普通复习

普通复习只复习到期卡，并计入 FSRS。

它会排除：

- 未到期卡。
- 已挂起卡。

### 提前复习

提前复习会复习当前卡组中的未挂起卡，不要求到期。

特点：

- 不计入 FSRS。
- 不改变卡片复习进度。
- 适合预习或临时过一遍。

### 全量复习

全量复习会按当前顺序复习所有未挂起卡。

特点：

- 不计入 FSRS。
- 不改变卡片复习进度。
- 普通全量复习不会复习挂起卡。

### 挂起全量复习

挂起全量复习会复习全部卡片，包括挂起和未挂起卡。

特点：

- 不计入 FSRS。
- 不改变卡片复习进度。
- 用于检查那些平时不参与复习的挂起内容。

### 微卡组复习

微卡组右键可以只复习某个微卡组：

- 复习此微卡组：按 FSRS，只复习到期且未挂起卡。
- 全量复习此微卡组：复习该微卡组内所有未挂起卡，不计入 FSRS。
- 挂起全量复习此微卡组：复习该微卡组内所有卡片，包括挂起卡，不计入 FSRS。

## 评分按钮与快捷键

显示答案后，可以用四个评分按钮：

- 重来。
- 困难。
- 良好。
- 简单。

默认快捷键：

```text
重来：Z
困难：X
良好：C
简单：V
```

显示答案默认快捷键：

```text
Space
```

这些快捷键都可以在设置中重新录制。

## FSRS 调度

普通复习使用 FSRS 调度。

设置项包括：

- 目标记忆保持率。
- 最大间隔天数。
- 启用间隔扰动。
- 启用短期学习步长。
- 学习步长。
- 重新学习步长。
- 每日新卡上限。
- 每日复习上限。

默认学习步长：

```text
1m, 10m
```

默认重新学习步长：

```text
10m
```

普通复习会写入复习记录，并更新卡片的 FSRS 状态。提前复习、全量复习、挂起全量复习不会写入 FSRS。

## 挂起

挂起用于让某些卡片临时退出正常复习系统。

挂起卡片不会出现在：

- 今日到期统计。
- 到期筛选结果。
- 普通复习。
- 提前复习。
- 普通全量复习。

挂起卡片可以出现在：

- 卡片列表中。
- 挂起全量复习中。
- 挂起全量复习此微卡组中。

取消挂起后，如果卡片已经到期，它会重新显示在到期卡中。

## 卡片详情与快速编辑

卡片可以查看详情并编辑：

- 正面。
- 背面。
- 类型。
- 来源。

复习时显示答案后，也可以进入编辑当前卡片，不需要退出复习再去列表中查找。

## 来源笔记与跳转

通过选中文本制卡时，插件会记录来源笔记。

支持：

- 打开来源笔记。
- 在复习显示答案后跳转到来源旁边现场修改。
- 卡片中的 Obsidian 双链可以点击跳转。
- 来源以 Obsidian 双链形式展示和编辑。

如果来源笔记不存在，插件会提示来源不存在。

## 双链跳转

EddieCards 会在卡片预览、卡片详情和复习界面中渲染 Obsidian 双链。

支持的形式包括：

```text
[[笔记名]]
[[文件夹/笔记名]]
[[笔记名#标题]]
[[笔记名|别名]]
```

点击卡片内容中的双链时，会在 Obsidian 笔记区打开对应笔记。

跳转规则：

- 普通双链会跳转到对应笔记。
- 带标题的双链会尽量跳转到对应标题位置。
- 带别名的双链显示别名，但跳转目标仍然是原始链接。
- 如果目标笔记不存在，会按 Obsidian 的解析结果处理；无法解析时不会破坏当前卡片。

来源字段也使用 Obsidian 双链语义。卡片详情中可以编辑来源，复习时显示答案后也可以打开来源，方便边复习边回到原笔记修改。

这个功能不会向原 Markdown 文件插入块引用或锚点，也不会污染原笔记。

## Markdown 渲染

卡片内容使用 Obsidian 的 Markdown 渲染能力。

支持：

- Markdown 文本。
- Obsidian 双链。
- 图片链接。
- LaTeX 数学公式。
- 代码块。

导出 Anki 时会尽量处理：

- 图片资源。
- Markdown 转 HTML。
- LaTeX 公式。

## AI 制卡

AI 制卡可以从当前笔记生成闪卡。

入口：

- 命令面板：`AI 从当前笔记生成闪卡`。
- 卡组右键菜单：`AI 制卡`。

设置项：

- Base URL。
- API Key。
- 模型。
- 检测模型。
- 手动模型名。
- AI 提示词模板。

模型下拉菜单可以通过当前 Base URL 和 API Key 检测得到。

AI 生成流程：

- 选择目标卡组。
- 选择卡片类型。
- 选择提示词模板。
- 输入或修改提示词。
- 实时查看生成过程。
- 生成后预览卡片。
- 勾选需要加入的卡片。
- 添加到指定卡组。

提示词模板支持：

- 新增模板。
- 删除模板。
- 修改模板名称。
- 修改模板内容。
- 下拉切换模板。

默认模板包括：

- 基础问答。
- 公式习题。

## 搜索、筛选与排序

卡片列表支持：

- 搜索正面、背面和来源。
- 按 Enter 执行搜索。
- 点击搜索按钮执行搜索。
- 清空搜索。

筛选项：

- 全部。
- 基础。
- 习题。
- 到期。
- 新卡。
- 学习中。

排序项：

- 最近创建。
- 最早到期。
- 复习最多。
- 最近修改。

## 健康检查

健康检查用于扫描卡片引用是否失效。

会检查：

- 来源笔记是否存在。
- 双链是否断开。
- 图片资源是否丢失。

健康检查不会自动修改卡片，只负责标出问题。

## 导出 Anki

卡组可以导出为 Anki `.apkg`。

导出特点：

- 每张 EddieCards 卡片导出为基础 Anki 卡。
- 微卡组在导出时相当于组织结构，不会成为 Anki 的特殊学习队列。
- Markdown 会转换为 Anki 可显示的 HTML。
- 图片会尽量打包。
- LaTeX 公式会尽量保留为 Anki 可渲染格式。

入口：

- 卡组右键菜单：`导出 Anki`。

## 备份与恢复

插件支持完整备份和恢复。

备份内容包括：

- 所有卡组。
- 所有卡片。
- 微卡组和学习顺序。
- 卡组文件夹。
- 插件设置。
- AI 提示词模板。

备份文件会导出为 JSON。

入口：

- 卡组右键菜单：`备份`。
- 卡组右键菜单：`恢复`。

恢复备份会覆盖当前 EddieCards 数据。

## 数据存储

插件数据存放在插件目录下：

```text
data/
```

卡组数据按卡组拆分存储：

```text
data/decks/
```

每个卡组有独立的数据文件，便于减少单个数据文件过大带来的维护压力。

导出文件默认存放在：

```text
data/exports/
```

## 推荐工作流

1. 在 Obsidian 笔记中学习或整理内容。
2. 选中一段 Markdown，用快捷键制成基础卡或习题卡。
3. 把相关卡片组成微卡组。
4. 通过拖拽调整概念、定理、例题、习题的顺序。
5. 使用普通复习处理到期卡。
6. 考前使用提前复习、全量复习或微卡组复习。
7. 对暂时不需要的卡片使用挂起。
8. 定期使用健康检查和备份。

## 适用场景

EddieCards 特别适合：

- 数学、物理、计算机等有强前置依赖的学科。
- 需要把概念、定理、证明、习题按顺序组织的学习场景。
- 希望在 Obsidian 内完成笔记、制卡、复习、修正的人。
- 希望保留 Anki 导出能力，但日常主要在 Obsidian 内复习的人。

## 注意事项

- 普通 FSRS 复习会改变卡片调度状态。
- 提前复习、全量复习、挂起全量复习不计入 FSRS。
- 挂起卡不会进入普通复习和普通全量复习。
- 如果来源笔记被删除，卡片仍然存在，但来源跳转会提示来源不存在。
- 双链定位依赖 Obsidian 的链接解析能力。插件会尽量跳转到对应笔记或标题，但不会主动污染原 Markdown 文件。
- AI API Key 只保存在本地插件数据中。
